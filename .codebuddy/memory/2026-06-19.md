### 修复：选中条目图标变蓝问题
- **根因**：Qt 在 `QTreeView` 选中状态下，`QStyledItemDelegate::paint` → `drawControl(CE_ItemViewItem)` 会用 `QIcon::Selected` 模式绘制图标，对 SVG 图标应用选中高亮色（蓝色）染色。仅设 `showDecorationSelected=false` 不够，因为图标本身仍被染色。
- **之前失败方案**：在基类 paint 之后用 Normal 模式重绘覆盖 — 不可靠，基类染色可能残留或位置不匹配。
- **最终方案**：在 `NoTintDelegate::paint` 中先 `initStyleOption` 取出 icon，清空 `opt.icon` 后由 style 绘制背景+文字（不含图标），再用 `QIcon::Normal` 模式自行绘制图标。彻底避免基类接触图标。

### 修复：运行时警告日志
- **QInputDialog 几何警告**（`QWindowsWindow::setGeometry: Unable to set geometry 153x38...`）：根因是 `MainWindow.cpp` 的 `inputDialogText()` 辅助函数在 `exec()` 前调用了 `show()`，show+exec 双重显示导致对话框用过小默认尺寸(153x38 < 最小 190x101)报警。修复：移除 `show()`，改在 exec 前显式 `setInputMode(QInputDialog::TextInput)` + `setMinimumSize(380,140)` + `resize(480,180)`。
- **QFileSystemWatcher 警告**（`FindNextChangeNotification failed for "..." (拒绝访问。)`）：`QFileSystemModel` 内部 watcher 在被监视目录变为不可访问（删除/权限变更/盘符移除）时的无害警告。修复：在 `main_filemanager.cpp` 与 `main.cpp` 两个入口 `qInstallMessageHandler` 安装过滤器，过滤此特定警告，其余消息照常转发默认处理器。
- 编译环境：MSVC 构建需先 `call vcvars64.bat` 设置 INCLUDE/LIB，否则 `type_traits` 等标准头找不到。VS 自带 cmake 路径：`C:\Program Files\Microsoft Visual Studio\18\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe`。
